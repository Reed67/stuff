Grundwissen OOP - Begriffe


Die objektorientierte Programmierung (OOP) ist ein auf dem Konzept der Objekt-orientierung basierendes Programmierdenkweise. Dabei sollen die existierenden Grundstrukturen des Programmiergegenstandes im Programm wiedergespiegelt werden. Das kann von Anwendungsfall zu Anwendungsfall zu unterschiedlichen Programm-strukturen führen. Dabei kommen Techniken der OOP zur Anwendung, die bereits beim Entwurf der Software beginnen und sich bis hin zum Refaktoring fortsetzen.

Ein Modell dieser Programmstrukturen wird in der Entwurfsphase i.d.R. Mit UML-Diagrammen (insbesondere Klassendiagrammen) aufgestellt. Dieses Modell enthält die Informationen über die zu verwendenden Objekte und deren Abstraktionen, ihre Typen.

Die Umsetzung dieses Paradigmas erfordert die Einführung verschiedener Konzepte, insbesondere Klassen, Abstrahierung, Kapselung, Vererbung, Polymorphie und spätes Binden.

Darunter wird hier folgendes verstanden:


Klassen
softwareseitige Darstellung eines real abzubildenden Objektes

mit diesen werden Instanzen (s.w.u.) erstellt


Abstrahierung
Objekte werden möglichst abstrakt modelliert und gehalten, sodass diese eine hohe Wiederverwendbarkeit haben


Kapselung
Bei der Kapselung wird alles was ein Objekt benötigt, in diesem Objekt selbst hinterlegt. Es kann also unabhängig von anderen Teilen des Programms verwendet werden.


Vererbung
unter Vererbung versteht man, dass von einer existierenden (Eltern-)Klasse abgeleitete (Kind-/Sub-)Klassen, Eigenschaften und Methoden der Elternklasse „erben“. Damit lässt sich doppelter Code vermeiden. Hierbei wird vom Algemeineren zum Spezielleren vererbt. Die Subklasse kann dem Objekt beliebige neue Eigenschaften und Methoden hinzufügen oder bestehende überschreiben.


Polymorphie
Eine Subklasse kann vererbte Eigenschaften und Methoden überschreiben, um sich zu spezialisieren. Geschieht dies so, dass durch Vererbung überschriebene Methoden an Stelle der Methoden des vererbenden Objektes ausgeführt werden, dann spricht man von Polymorphie. Polymorphie ist eine Möglichkeit, einer durch ähnliche Objekte ausgeführten Aktion einen Namen zu geben, wobei jedes Objekt die Aktion in einer für das Objekt geeigneten Weise implementiert.


Instanzen
Sie sind die während des Programmablaufes aus den definierten Klasse entstehenden Programmobjekte.


Methoden
Methoden beschreiben das Verhalten eines Objektes. Auch wenn es sich bei JavaScript genaugenommen nicht um die üblicherweise verstandenen Methoden handelt.


Eigenschaften
Eigenschaften sind die Variablen eines Objektes.


Spätes Binden

Spätes Binden bindet einen Methodenaufruf erst zur Laufzeit an eine bestimmte Methode.



Objekte in JavaScript – Grundlagen und Notation


Grundsätzlich ist alles in JavaScript ein Objekt. Auch für einfache Typen gibt es entspre-chende Objekte. Einige Typen (Datum, Funktion) existieren nur als Objekt. Alle Objekte haben einen eineindeutigen case-sensitiven Namen. Objekte sind hierarchisch angeordnet und innerhalb dieser Hierarchie sind auch weitere folgende Unterobjekte, Methoden und Eigenschaften eineindeutig benamt.

Bei der Notation von JavaScript gilt dabei Folgendes:

  • für die Notation gelten alle Regeln der allgemeinen JavaScript-Notation (Siehe dazu: https://academy2.unister-gmbh.de/mod/resource/view.php?id=3376),

  • es wird bei einem Wurzelobjekt begonnen zu notieren,

  • folgende Unterobjekte, Methoden oder Eigenschaften werden mit „.“ voneinander getrennt,

  • Methoden werden mit abschließendem „()“ notiert und

  • Parameter für Methoden werden innerhalb der runden Klammern übergeben und mit „,“ voneinander getrennt.

Array


Beschreibung


Das Array-Objekt speichert eine Gruppe von Daten in einer Variablen. Die Elemente innerhalb des Arrays können unterschiedliche Typen haben und es wird über einen Index auf diese zugegriffen.

Ein Element kann ebenfalls ein weiteres Array speichern – dies führt zu mehrdimen-sionalen Arrays. Dabei wird für jedes Teilarray ein eigener Index verwendet. Zweidimensionale Arrays stellen Tabellen dar, mehr als 2 Dimensionen sind Bäume.

Array-Objekte können auch assoziative Arrays sein. Dabei werden die Indizies zu eineindeutigen Stringvariablen. In JavaScript kann sich jedes Objekt wie ein assoziatives Array verhalten.

Boolean


Beschreibung

Boolean speichert boolsche Werte als Objekt. In JavaScript, repräsentieren die Keyworte true und false primitive boolean Werte. Auch wenn der Datentyp boolean dieser primitiven Werte boolean ist, sind diese nicht Instanzen des Boolean-Objekts.

Der JavaScript-Interpreter verwendet kein Boolean-Objekt für die implizite Konvertierungen von Datentypen (Umwandlung von Nicht-Boolean-Werte in einer if-Anweisung), es kann aber das Boolean-Objekt als eine Funktion verwendet werden, um einen Wert explizit in ein primitiven boolean-Wert zu konvertieren. Wenn der JavaScript-Interpreter, einen primitiven boolean-Wert in ein Objekt konvertieren muss, wird das Boolean-Objekt verwendet.

Um eine Boole'sche Konstante mit dem Wert true zu initialisieren, notieren Sie hinter dem frei vergebbaren Namen der Konstante die Objektfunktion new Boolean(). Als Parameter übergeben Sie der Funktion entweder den Wert true (ohne Anführungszeichen) oder eine beliebige Zeichenkette.

Um einen booleschen Wert mit false zu initialisieren, wird false, gar nichts, eine leere Zeichenkette, „0“, „null“ (ohne Anführungszeichen) oder „undefined“ übergeben.

Object


Beschreibung


Object ist das Basisobjekt für alle JavaScript-Objekte. Das Object-Objekt erbt alle seine Eigenschaften und Methoden von Function.prototype und dem Object.prototype-Objekt, mit Ausnahme der prototyp-Eigenschaft, die auf das Object.prototype-Objekt zeigt.

Objekte, die in JavaScript instanziiert werden können, erben vom Function.prototype-Objekt, alle anderen erben vom Object.prototype-Objekt.

Es kann ein beliebiges Objekt mit dem Konstruktor des Object-Objektes erstellt werden. Wird der Konstruktor der Object-Objekt ohne Parameter oder mit „null“ oder „undefined“ verwendet, so ist das erzeugte Objekt vom Typ Object. Wird der Konstruktor mit einem primitiven Datentyp aufgerufen, dann wird der Typ des erstellten Objekts vom Typ des primitiven Datentyps übernommen: Boolean bei primitiven boolean-Wert, Number im Falle einer primitiven Zahl, String im Falle eines primitiven Strings. Wird der Konstruktor mit einem Objekt aufgerufen, so ist der Typ des erzeugten Objekts gleich dem entsprechenden Objekt.

Beschreibung

Number


Beschreibung

Number stellt einen numerischen Wert als Objekt dar. Das Number-Objekt unterstützt numerische Konstanten und Methoden zur Arbeit mit Zahlen. Primitive numerische Werte (z.Bsp. 1, 2, 33.33, -2.3456) sind nicht gleich dem Number-Objekt, deren Typ ist Number und nicht Object. Wenn mit Zahlen in JavaScript gearbeitet wird, wird mit dem primitiven numerischen Wert gearbeitet, nicht mit Instanzen des Number-Objekts.

Der JavaScript-Interpreter verwendet keine Number-Objekt für die implizite Konvertierungen von Datentypen (wenn ein nicht-numerischen Wertzu einem numerischen konvertiert werden muss, zum Beispiel 2 * "32"), aber die Anzahl Objekt kann als eine Funktion verwendet werden, um eine Umwandlung Wert auf einen primitiven numerischen Wert explizit. Das DOM


Einleitung

Das DOM (Document Object Model) ist die Spezifikation einer Schnittstelle für den Zugriff auf HTML- oder XML-Dokumente. Es wird vom World Wide Web Consortium (W3C) definiert und eine Implementierung besteht im Sinne der objektorientierten Program-mierung aus einem Satz von Klassen zusammen mit deren Methoden und Attributen. Diese erlauben es, dynamisch den Inhalt, die Struktur und das Layout eines Dokuments zu verändern.

Struktur

Die Elemente des DOM sind in einer Baumstruktur angeordnet. Für das DOM bildet das aktuelle Fenster (window) die Basis des Objektmodells. Für dieses Dokument sind die Objekte:

navigator

location

history und

screen von Bedeutung.

Das (window.)document-Objekt stellt die Wurzel des dargestellten HTML-Dokumentes dar und wird mit seinen Objekten in einem Extraabsatz beschrieben.

Über das navigator-Objekt, welches unterhalb des window-Objektes liegt, kann in einem JavaScript Informationen darüber ermittelt werden, welchen Web-Browser der Anwender verwendet und es können zusätzliche Angaben zum verwendeten Browser bestimmt werden. document


Das document-Objekt stellt das im Browser geladene Dokument dar. Per Definition besteht dieses Dokument aus einer Anzahl unterschiedlich ansprechbaren „Collections“ (Sammlungen)und unterschiedlichen Knoten, welche unterschiedliche Typen haben. Als Collectionen stehen zur Verfügung:

  • anchors

  • applets

  • forms

  • images und

  • links.

Diese Collectionen sind Arrays und können über document['collName'] angesprochen werden. Die Notation document['images'] enthält dann alle im Dokument vorhanden Bilder. Über den Index kann dann ein einzelnes Element der Collection adressiert werden (z.Bsp. document['images'][0]).

Als Knoten (nodes) stehen folgende Typen zur Verfügung:

  • Dokumentknoten (stellt die gesamte Baumstruktur dar),

  • Dokumentfragmentknoten (einen Teil der Baumstruktur des Dokuments),

  • Elementknoten (exakt ein Element in HTML oder XML),

  • Attributknoten (exakt ein Attribut in HTML oder XML) und

  • Textknoten (textueller Inhalt eines Elements oder Text zwischen Elementen).

Attributknoten sind eine besondere Knotenart, denn sie kommen nicht als Knoten in der Baumstruktur vor, die vor allem durch Elementknoten gebildet wird. Attributknoten sind demnach keine „Kinder“ von Elementknoten, sondern Eigenschaften von ihnen.

Die Adressierung von Knoten erfolgt über das Ansprechen eines Elementes mit einer Id, mit einem Tag-Namen und der Adressierung der Beziehung - Elternknoten als parent, Kindknoten als child und Geschwisterknoten als sibling.
